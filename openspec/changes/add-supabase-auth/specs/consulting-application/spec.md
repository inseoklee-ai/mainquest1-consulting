## MODIFIED Requirements

### Requirement: 내 신청 내역 조회

시스템은 신청자가 자신의 신청 내역을 조회할 수 있어야 한다(SHALL). 본인 확인 방식은 로그인 여부에 따라 다르다.

- **비로그인 조회**: 신청할 때 쓴 **이메일 + 휴대폰번호**를 모두 입력해야 하며, 두 값이 모두 일치하는 신청만 보여준다. 타인의 신청 내역은 노출하지 않는다.
- **로그인 조회**: 로그인한 사용자는 별도 입력 없이, **Supabase가 검증한 로그인 계정의 이메일**과 일치하는 신청 내역을 볼 수 있다. (이메일 소유가 이미 인증되었으므로 휴대폰 재입력을 요구하지 않는다.)

두 경우 모두, 입력/계정 기준과 일치하지 않는 타인의 신청은 보여주지 않는다.

#### Scenario: 본인 신청 내역 조회 (비로그인)
- **WHEN** 비로그인 사용자가 자신이 신청할 때 사용한 이메일과 휴대폰번호를 입력해 조회한다
- **THEN** 시스템은 두 값이 모두 일치하는 신청 내역(신청 일시, 신청한 주 등)을 보여준다

#### Scenario: 신청 내역이 없는 경우 (비로그인)
- **WHEN** 신청한 적이 없는 (이메일 + 휴대폰번호) 조합으로 조회한다
- **THEN** 시스템은 신청 내역이 없음을 안내한다

#### Scenario: 타인 내역 비노출 (비로그인)
- **WHEN** 입력한 이메일과 휴대폰번호가 서로 다른 신청자들의 값과 부분적으로만 일치한다(예: 이메일만 같고 휴대폰번호는 다름)
- **THEN** 시스템은 두 값이 모두 일치하는 신청만 보여주며, 일치하지 않으면 내역을 노출하지 않는다

#### Scenario: 로그인 사용자의 내 신청 조회
- **WHEN** 로그인한 사용자가 "내 신청 내역"을 연다
- **THEN** 시스템은 휴대폰 재입력 없이, 로그인 계정의 검증된 이메일과 일치하는 신청 내역을 보여준다

#### Scenario: 로그인했지만 신청 이력이 없는 경우
- **WHEN** 로그인한 사용자의 이메일로 접수된 신청이 하나도 없다
- **THEN** 시스템은 신청 내역이 없음을 안내한다
