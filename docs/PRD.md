# 기획 문서 (PRD) — 중소기업 AI 도입 컨설팅 신청 서비스

## 한 문장 (도메인 연결)

> **내 전문 분야인 '중소규모 회사 운영'의 이 불편을** — 대표 한 사람이 모든 일을 책임지지만 인적·물적 자원은 늘 부족한 현실을 — **AI 도입으로 극복하도록, 부담 없이 컨설팅을 신청할 수 있게 만든다.**

## 왜 나인가 (Author's Domain)

반도체 엔지니어로 일한 뒤 **해외 통신망 설치회사를 직접 운영**했다. 그 과정에서 중소규모 회사의 가장 큰 현실을 몸으로 겪었다: **대표가 영업·운영·관리·기술까지 모든 것을 혼자 책임져야 하고, 일은 대기업만큼 많은데 능력 있는 인재도 자금도 충분치 않다.** 지금 AI가 생산성 향상과 운영 효율화에 실질적인 힘이 되는 것을 보며, **같은 어려움을 겪는 중소기업이 AI를 도입하도록 돕고 싶다.** 중소기업의 핵심 애로사항인 **자원(인적·물적) 부족**을, AI 도입이 가장 현실적으로 메울 수 있다고 확신한다.

## 서비스 개요

중소기업 대표가 **로그인·회원가입 없이** 이메일·휴대폰번호만 남기면 "AI 도입 컨설팅"을 신청할 수 있는 웹 서비스. 상담 자원이 한정돼 있으므로 **주당 정원(4명)** 안에서 선착순으로 접수하고, 신청자는 자신의 신청 내역을 언제든 조회할 수 있다.

## 타겟 사용자

- **1차 타겟**: 인력·자금이 부족해 AI 도입을 "어디서부터 시작할지" 막막해하는 **중소기업 대표 / 실무 책임자**
- **특징**: 바쁘고, 가입 절차에 시간을 쓰기 싫어하며, 확실한 정원·일정 안내를 원함
- **사용 맥락**: 홍보 링크·소개를 통해 유입 → 최소 입력으로 즉시 신청 → 내 신청 상태 확인

## 해결하려는 문제

| 문제 | 이 서비스의 해결 |
|------|------------------|
| 대표가 모든 걸 혼자 처리, 시간이 없다 | 회원가입 없이 **2개 항목(이메일·휴대폰)** 으로 30초 신청 |
| AI 도입을 어디서 시작할지 막막하다 | "컨설팅 신청"이라는 명확한 첫걸음 제공 |
| 상담 자원이 한정적이라 과부하 위험 | **주당 4명 정원**으로 감당 가능한 만큼만 접수 (초과 절대 불가) |
| 신청했는지, 언제인지 헷갈린다 | **내 신청 내역 조회** 제공 |

## 핵심 기능 (스펙 요약)

1. **신청** — 이메일·휴대폰번호로 로그인 없이 접수 (형식 검증)
2. **중복 차단** — 동일 (이메일+휴대폰) 조합은 전체 1회만
3. **주 단위 정원** — 한 주 최대 4명, 동시 신청에도 초과 없음
4. **내 신청 내역 조회** — 이메일+휴대폰 둘 다 일치할 때만 조회

> 상세 요구사항·시나리오: [openspec 스펙](../openspec/changes/add-consulting-application/specs/consulting-application/spec.md)

## 기술 스택

- **Python + FastAPI + SQLite** — 선정 근거: [design.md `D0`](../openspec/changes/add-consulting-application/design.md)
- (심화) **Supabase** 로 회원가입·로그인 및 신청을 '내 것'으로 연결

## 성공 기준 (이번 과제 범위)

- 브라우저에서 신청 → 내 내역 조회가 **실제로 동작**
- 주 4명 정원이 **초과되지 않음** (동시성 포함)
- GitHub 저장소 + 배포 URL 로 확인 가능

## 범위 밖 (Non-Goals)

- 결제, 관리자 대시보드, 알림(이메일/SMS) 발송
- 정원(4명)을 운영 중 변경하는 기능 — 이번엔 고정
- 대기(waitlist) 기능
